About the Project

About This Project



INTRODUCTION

Crimes are a common social problem affecting the quality of life and the economic growth of a society. It is considered an essential factor that determines whether or not people move to a new city and what places should be avoided when they travel. With the increase of crimes, law enforcement agencies are continuing to demand advanced geographic information systems and new data mining approaches to improve crime analytics and better protect their communities.

OUR GOAL

For this project, we use data mining and visualization tools to figure out the most common crime location and time. Using regression, we try to predict the probability that a crime would happen with provided weather, location and time data. By providing a data mining approach to determine the most criminal hotspots and find the type, location and time of committed crimes, we hope to raise people’s awareness regarding the dangerous locations in certain time periods.

Our Dataset

What Our Data Contains

Key informaion related to crime location, date, type and weather at the time period.


We obtain the crime information from CHICAGO DATA PROTAL website, which is an open data source that contains information about Chicago communities. In order to further explore what might affect the probability of a crime, we also include some information about weather of Chicago at the corresponding time period. The two set of data are combined together for detailed analysis. The attributes we selected in order to explore crime patterns for our project:

　
  • Date: Date when the incident occurred (e.g. 12/31/2015 11:23:00 PM).
  • Crime Type: The primary description of the IUCR code (the Illinois Unifrom Crime Reporting code). Some major crime types appearred in this dataset include: theft, robbery, assault, criminal damage etc.
  • Location Description: Description of the location where the incident occurred. Some major locations appearred in this dataset include: restaurant, street, apartment, parking, school district, retail store etc.
  • District: Indicates the police district where the incident occurred. See the districts at https://data.cityofchicago.org/d/fthy-xz3r.
  • Block: The partially redacted address where the incident occurred, placing it on the same block as the actual address.
  • Longitude & Latitude: The longitude and latitude of the location where the incident occurred. This location is shifted from the actual location for partial redaction but falls on the same block.
  • Hourly Visibility: This shows the hourly visibility represented by an integer ranging from 0 to 10, which indicates the visibility of the time period.
  • Temperature: The temperature of the time period, which is reported by the Chicago weather station.
  • Wind Speed: This attribute describes the wind speed of the time period, represented by an integer that indicates the level of wind speed.




Analysis

Crime Types, Location and Time

Using visualization to better understand the crime patterns

From the bubble chart visualization of crime location description, we discovered that most crimes happened on streets, residence house, sidewalks and parking lots. Some other locations like retail stores, restaurants, department stores are also very likely to have crimes. This may suggest that given a certain time period and under same condition, the probability of a crime would happened on a major street may be larger than the probabilityt that a crime would happen at a school district. Given this information, although we can not conclude that crimes are more likely to happen at which location, we can say that the type of the location is closely related to the crime rates and we can use it as a significant feature for our regression algorithm.



Location on map and crime types

The most common crime types include: theft, criminal damage, narcotics, burglary, robbery and deceptive practice.

Our Steps

Data Cleaning

The original data was cleaned and analyzed by Dataiku DataScienceStudio to eliminate any blank or invalid rows and columns. Over 80 kinds of location_type data was then manually categorized and reduced to 20 as we group similar types. In the end, only useful column like date, location, crime type and weather condition were kept in the csv file. The original dataset includes domestic crimes that happened indoor, which are not relevant to our project. Thus, those types of crimes were excluded from the dataset.

Data Processing

The original dataset does not contain all the attributes we want and is not well formated. The separated dataset was merged together and we use several statistical tools to manipulate some of the attributes in order to conduct regression.

COMBINE THE DATA



We get the crime rate of each district and add that information to the original dataset because we are interested to know the reported crime rate of these areas. When merging the data, we chose several attributes from the Chicago weather dataset including "temperature", "wind speed" and "hourly visibility". With all these information, we are able to conduct regression to predict the crime rate.

MODIFY DATA FORMAT



Some of the original data format is not useful for our analysis and regresssion algorithm. For example, the "Date" of the crime is recorded with the normal date format which includes and date and time like 10/31/2015 10:23:00 PM. When we conduct regression, we try to find the pattern between the time period of the day and the crime rate. Therefore, we change the data format into "time of day" ranging from 0 - 23, which makes it easier to explore pattern between time of day and crime rate.

USING K MEANS


We use the k-means clustering machine learning method to explore the data. K-means groups similar objects into the same group. In this way, it’s a much efficient way to extract information from a very large dataset and find similarities among the features of the crime record.

Regression

We use multiple regression model to calculate the relationship between crime rate and its relative features in R. Here’s the command: fit = glm(chicago$Crime~factor(chicago$Date) + factor(chicago$Location_Description) + factor(chicago$District)+ chicago$HourlyVisibility + chicago$Temperature + chicago$WindSpeed)
